以下のお題について Morphological Analysis（形態分析）で組合せ空間を構築し、異質な組合せを抽出してください。あなた（このメインセッション）が **lead = 次元設計 + 組合せ生成・キュレーション役** です。

# お題

$ARGUMENTS

# 重要な前提

これは **組合せ空間で発散させる** プロトコル。`/idobata:brainstorm` がフラットなアイデア生成、`/idobata:scamper` が既存物の変形に対し、本プロトコルは **空間全体を構造化** してサンプリングする。

- **判定・採否を出さない**
- **critic 役を置かない**
- 出力は組合せ空間の構造 + 抽出された異質組合せ集合

適用領域の例（網羅ではなく代表例）:
- Web サービス・アプリのコンセプト: ターゲット × 提供形態 × 課金モデル × コア機能 × 差別化軸
- プロダクト企画: 素材 × 形状 × 機構 × 用途 × 価格帯
- 実験計画: 因子 × 水準の網羅
- サービスデザイン: タッチポイント × 提供形態 × 課金 × 期間
- 業務プロセス設計: トリガ × 担当 × 工程 × 引き継ぎ × 完了条件
- 制度・ルール設計: 範囲 × 強制度 × ペナルティ × 例外
- 物語・キャラ設計: 性格 × 出自 × 動機 × 障害 × 変化アーク
- 任意の「次元 × 値」で構造化できるお題

# プロトコル

## Step 1: 次元分解（dimension-proposer 1〜2 個並列）

`dimension-proposer` を 1 個 spawn（複雑なお題なら異なる lens で 2〜3 個並列も可）。

入力:
- お題本文
- 期待される次元数の目安（5〜8）

出力: 次元 5〜8 個 + 各次元の代表値 + 直交性チェック。

複数 spawn した場合は lead が次元を統合・直交化する。

## Step 2: 値生成（value-generator を次元数ぶん並列）

確定した N 次元それぞれに対し `value-generator` を 1 個 spawn し、合計 **N 個並列**。

各 spawn prompt:
- お題本文（全体コンテキスト）
- **その value-generator が担当する 1 次元のみ**（次元名 + 意味）
- 既知の凡庸値（避けるべきもの）

出力: 各次元の値候補 10〜20 個（カテゴリ別）。

## Step 3: 形態箱（Morphological Box）構築（lead 単独）

全次元 × 全候補値の表（Zwicky Box）を構築:

```
| 次元 \ 候補 | v1 | v2 | v3 | ... |
| 次元 1     | a1 | a2 | a3 | ... |
| 次元 2     | b1 | b2 | b3 | ... |
| 次元 3     | c1 | c2 | c3 | ... |
| ...        |    |    |    |     |
```

組合せ空間サイズ = ∏(各次元の候補値数)。通常 1000〜100000 オーダーになる。

## Step 4: クロス整合性マトリクス（任意 / 中規模以上で推奨）

次元 i × 次元 j の値ペアで「ありえない / 矛盾する」組合せをマーク。
全ペアでフィルタすることで実質空間サイズを大幅削減できる（Zwicky の元手法ではこれが重要）。

軽量運用ではこのステップをスキップして Step 5 で直接サンプリングしてもよい。

## Step 5: 異質組合せのサンプリング（lead 単独）

組合せ空間から **異質性を最大化** するように N_sample = 20〜50 個の組合せを抽出する。
全数列挙ではなく **戦略的サンプリング** を行う:

1. **既知の組合せ**（既に使われている / 凡庸） を 3〜5 個まず明示し、以降のサンプリングではこれらを避ける
2. **エッジ組合せ**: 各次元の極端値同士の組合せ（最小 × 最大 など）
3. **ランダム組合せ**: 一様乱択 5〜10 個
4. **wildcard 組合せ**: 「突飛・実験的」カテゴリの値を必ず 1 つ含む組合せ
5. **直交ペア駆動**: 既知から各次元 1 つずつ値を変えた近傍組合せ（ablation）

## Step 6: 組合せの命名と意味付け（lead 単独）

抽出した各組合せに対し:
- 短い名前（人間が記憶しやすい）
- 1〜2 文の説明（その組合せが表すコンセプト / 期待される性質）
- 期待される強み・想定される弱み（弱みを書いてよいが「ダメ」判定はしない）

## Step 7: 最終出力

```
# Morphological Analysis 結果

## お題
<元お題>

## 次元構成
1. <次元1>: <意味> / 候補数 = M1
2. <次元2>: <意味> / 候補数 = M2
...
全組合せ空間サイズ = ∏Mi = N_total

## 形態箱（Morphological Box）
（Step 3 のテーブル）

## 既知 / 凡庸組合せ（除外対象）
- <組合せ>: 既存の何に相当するか

## 抽出された異質組合せ（N_sample 個）

### 組合せ A: <名前>
- 値の選択: 次元1=v, 次元2=v, ...
- コンセプト説明: ...
- 期待される強み: ...
- 想定される弱み: ...
- 抽出戦略: <エッジ / ランダム / wildcard / ablation>

### 組合せ B: ...
（N_sample 個）

## 次の収束ステップ提案（お題ドメインに応じて選ぶ）
- これらの組合せを /idobata:pugh-matrix で datum=既知組合せ として比較
- 上位を /idobata:rational-choice で重み付き定量評価
- 特定 1 案を /idobata:red-blue で攻撃検証
- ドメイン固有の評価手段（実装試作・実験・実データ検証・ユーザテスト等）に渡す
```

# 使用例

本プロトコルは組合せ的に膨大な空間を持つ任意のお題に適用できる。以下は適用例の一部:

## 例: Web サービス・アプリのコンセプト
- 次元: ターゲット顧客 / 提供形態 / 課金モデル / コア機能 / チャネル / 差別化軸
- 値例:
  - ターゲット = 個人 / 小規模事業者 / 大企業 / 公共機関 / 開発者 / シニア層
  - 提供形態 = Web / モバイル / API / デスクトップ / ハードウェア / Slack bot
  - 課金 = 無料 / サブスク / 従量 / 広告 / 売り切り / 寄付
  - コア機能 = 記録 / 共有 / 分析 / 自動化 / 推薦 / マッチング

## 例: プロダクト企画
- 次元: 素材 / 形状 / 機構 / 用途 / 価格帯 / 販売チャネル
- 値例: 素材 = 紙·木·樹脂·金属·布·食品 / 機構 = 折り畳み·伸縮·磁着·分離·発光

## 例: 実験計画
- 次元: 因子（独立変数）× 水準（離散値）の網羅的因子配置
- 既知の凡庸組合せ ＝ 既存の実験条件、として未踏領域をサンプリング

## 例: 制度・ルール設計
- 次元: 適用範囲 / 強制度 / 違反時ペナルティ / 例外条件 / 評価期間 / レビュー主体
- 値例: 全員·選択制·役職限定 / 罰金·差し戻し·公表·無し など

## 例: ストーリー・キャラクタ設計
- 次元: 性格軸 / 出自 / 動機 / 障害 / 関係性パターン / 変化アーク

# 重要な制約

- 次元設計を妥協しない（直交性が崩れると以降のすべてが歪む）
- value-generator を **1 spawn = 1 次元** に厳守（複数次元を兼任させない）
- lead は組合せの「良し悪し」を判定しない。整理・命名・サンプリング戦略のみ
- N_sample が 50 を超えるなら Step 4 のクロス整合性フィルタを必ず通す
